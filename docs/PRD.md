# mirror-vn PRD — 리빌드 (거울을 보지 마세요)

> 원작: 카카오페이지 웹소설 「괴담에 떨어져도 출근을 해야 하는구나」 수록 괴담 「거울을 보지 마세요」 (Qterw-D-718) 팬 창작 게임
> 이 문서는 게임의 **모든 판정 규칙의 단일 출처(single source of truth)**다. 코드는 이 문서의 테이블을 데이터로 옮긴 것이어야 하며, 문서에 없는 규칙은 구현하지 않는다.
> 관련 티켓: YEO-597 (리빌드 부모), YEO-598 (이 문서)
> 확정일: 2026-07-07

---

## 1. 핵심 원칙

1. **시점 — 1인칭.** 유저는 엘리베이터 안에서 거울(거울 속 존재)을 직접 보고 있다. CCTV 관찰 시점이 아니다. 스캔라인·타임스탬프 등 CCTV 마커 금지.
2. **난수 0%.** 모든 판정은 각본 테이블 기반. `Math.random` 판정 금지.
3. **규칙 텍스트 = 실제 판정.** `[시스템]` 의식서 조항에 적힌 숫자와 실제 판정값은 반드시 일치하거나, 단계화(경고→발동)로 정합성을 유지한다.
4. **억울한 죽음 금지.** 인식 지연·오인식이 사망 원인이 되지 않도록 모든 카메라 판정에 완충(감지 기준 타이머, 그레이스, 히스테리시스)을 둔다. 죽음의 원인은 항상 유저의 선택 또는 지연이어야 한다.
5. **몰입 우선 UI.** 명령형 튜토리얼 팝업 금지. 안내는 ①캘리브레이션(게임 밖) ②`[시스템]` 의식서 라인 ③인식 상태 인디케이터의 3겹으로만 제공.
6. **원작 지식 전제.** 팬 게임으로서 원작을 아는 유저가 유리한 설계(귀신 질문의 진실 등)는 의도된 것이다. 인트로에 별도 힌트를 심지 않는다.

## 3. 게임 플로우

```
CALIBRATION (게임 밖: 웹캠 권한 + 가위·바위·보 각 1회 인식 확인)
→ SCENE_01 진입 → SCENE_02 호출 → SCENE_03 의식 진입 (의식서 조항 표시)
→ PHASE_1 (RPS ×최대 5라운드, 1F→12F)
→ PHASE_2_ENTRY (진입 연출 = Phase 1 패배 수 차등)
→ PHASE_2 (귀신 주도 RPS ×3라운드, 라운드 사이 귀신 질문 Q1~Q3)
   ├─ R1 패배 → Q1 → R2 패배 → 인사이트 → Q2 → ROUND3_SETUP (펜+고무줄) → R3
   ├─ R3 가위 고정 → 의식 파탄 → WIN_CUTSCENE
   └─ (실패 분기 → Bad/Dead Ending, §5.2·§7)
→ ESCAPE_FROST (서리 닦기) → ESCAPE_HOLD (버튼 홀드, 12F→1F)
→ TRUE_ENDING_QUESTION (보상 질문 0~1회) → 엔딩 트랜스크립트 → 크레딧/retry
```

모든 씬 전환은 FSM 테이블에만 존재한다 (YEO-599). 엔딩 씬은 터미널 상태 — 진입 후 다른 트리거 발동 불가.

## 4. RPS 각본 테이블

### 4.1 Phase 1 — 거울 속 나 (1F→12F, 최대 5라운드)

귀신(거울 속 나)의 수는 **"유저를 이기는 손, 단 가위 불가. 가위가 필요하면 미러링"** 규칙으로 결정된다. 결정론적이며, 관찰하면 "가위를 절대 안 낸다"는 단서가 여기서부터 쌓인다.

| 유저 손 | 귀신 손 | 판정 | 효과 |
|---------|---------|------|------|
| ✋ 보 | ✋ 보 (가위 못 내므로 미러링) | 비김 | 층수만 상승 |
| ✊ 바위 | ✋ 보 | 패배 | 귀신 1단계 근접 |
| ✌ 가위 | ✊ 바위 | 패배 | 귀신 1단계 근접 |

- 종료: 4패 **또는** 5라운드 소진 → PHASE_2_ENTRY
- 층수 진행: 1F → 5F → 7F → 10F → 12F (라운드당 1단계)

### 4.2 Phase 2 — 귀신 주도 (3라운드)

의식서 조항 그대로 "응하지 않을 시 판정승을 넘긴다". 판정의 주도권이 귀신에게 있다.

| 라운드 | 유저 손 | 귀신 손 | 판정 | 비고 |
|--------|---------|---------|------|------|
| R1 | 아무거나 | ✊ 바위 | **패배 (각본 확정)** | 힌트 없음. 패배 후 Q1 |
| R2 | 아무거나 | ✊ 바위 | **패배 (각본 확정)** | 패배 후 인사이트 → Q2 |
| R3 | ✌ 가위 (펜+고무줄 고정) | 미러링 시도 → 실패 | **승리 (의식 파탄)** | WIN_CUTSCENE으로 |
| R3 | 가위 외 손 / 미응답 | — | 실패 | Bad Ending (§7) |

- R3 승리는 RPS 판정승이 아니라 **의식 파탄**이다: 거울 속 존재는 반사상이므로 고정된 가위를 미러링해야 하지만 불가능 → 존재/의식이 붕괴. ROUND3_SETUP 독백이 이 인과를 명시한다 (결정 2).
- 라운드 기록 텍스트(§5.4)는 Phase 1·2 전체의 내 손/귀신 손을 상시 노출한다.

### 4.3 귀신 질문 (Phase 2 라운드 사이)

| 질문 | 출처 | 진실 (통과) | 거짓 (즉사) |
|------|------|------------|------------|
| Q1 "이름이 뭐야?" | 원작 | 주인공 실명 | 가짜 이름 |
| Q2 "생일이 언제야?" | 원작 | 실제 생일 | 가짜 생일 |
| Q3 "지금 무서워?" | 게임 오리지널 | **긍정 ("응")** | 부정/허세 |

- 선택지는 **진실/거짓 2택**. 회피 선택지 없음.
- 응답 제한시간 초과 → 즉사 (Dead Ending). 제한시간은 `[시스템]` 라인으로 사전 안내 (예: `※ 10초 안에 대답하지 않으면 — 명복을 빈다.`)
- 진실 판별은 원작 지식 전제 (핵심 원칙 6). 인트로에 별도 심기 없음.

## 5. 인터랙션 판정 스펙

### 5.1 손 소실 (JUMP_SCARE 트랙) — 결정 6

| 시점 | 이벤트 |
|------|--------|
| 손 소실 3.0s | 경고 연출 (거울 일렁임 / 귀신 근접 반단계 / 저음) |
| 손 복귀 | 경고 해제, 타이머 리셋 |
| 손 소실 5.0s | JUMP_SCARE 발동 → 엔딩 |

- 적용 씬: RPS 진행 중만 (FSM에 허용 씬 명시). 인트로·컷씬·엔딩·질문 씬에서는 비활성.

### 5.2 Dead Ending 반응 판정 — 결정 5

- 카운트다운 시작 = **유저 손이 프레임에 감지된 시점** (인식 지연 미포함)
- 반응 제한: 1.5s (파라미터)
- 절대 상한: 신호 발생 후 4.0s — 손이 끝내 감지되지 않으면 발동 (손 숨기기로 무한 유예 불가)

### 5.3 ESCAPE_HOLD — 결정 7

- **판정**: 검지 끝(landmark 8)의 화면 좌표가 버튼 히트박스 안. 손 모양(포즈)은 판정하지 않는다.
- **시각 피드백**: 손끝이 히트박스에 들어오는 순간 버튼이 눌리며 발광, 닿아 있는 동안 유지 — "지금 인정되고 있다"가 항상 보인다.
- **진행도**: 층수 디스플레이 하강 (12F→1F). 프로그레스 바·시스템 텍스트 금지.
- **오인식 완충 4종**: ① 히트박스 히스테리시스 (진입 반경 < 이탈 반경) ② 좌표 EMA 스무딩 ③ 500ms 미만 드랍 무시 ④ 닦기→홀드 전환 그레이스 1.0s
- **실패 루프**: 유효 이탈 1회 = 하강 정지 + 귀신 1단계 근접 + 사운드 스팅 (복구 시 재개). 유효 이탈 누적 3회 = Bad Ending.

### 5.4 안내 3겹 구조

| 겹 | 위치 | 내용 | 톤 |
|----|------|------|-----|
| 1 | 캘리브레이션 (게임 밖) | 웹캠 권한 + 가위·바위·보 각 1회 인식 확인 | 직설적 UI 허용 |
| 2 | `[시스템]` 의식서 라인 | 규칙 전달 (가위바위보 하라, 3초, 열림 버튼, 질문 제한시간) | 독백·나레이션과 구분되는 서체/등장 연출 |
| 3 | 인식 인디케이터 | 내 손 감지 여부 + 인식된 손 모양만 최소 표시 | 판정 투명성 장치. 거울 프레임 언저리 |

라운드 기록 텍스트(내 손 vs 귀신 손, 텍스트만)는 3겹과 별도로 RPS 씬에서 상시 노출.

## 7. 엔딩 분기

| 엔딩 | 조건 |
|------|------|
| **True Ending** | ESCAPE_HOLD 완주 → 보상 질문 (0~1회, LLM 진실 답변) → 자동 엔딩 진행 |
| **Bad Ending (빙의)** | Phase 2 R3 실패 · ESCAPE_HOLD 유효 이탈 3회 |
| **Dead Ending** | 귀신 질문 거짓 대답 · 응답 시간 초과 · Phase 2 신호 미반응 (§5.2) |
| **Jump Scare** | RPS 중 손 소실 5초 (§5.1) |

- True Ending: 질문은 최대 1회. "안 묻고 떠난다" 선택도 유효한 True Ending. 답변 타자기 출력 완료 → `*하자. 할 수 있다.*` → 트랜스크립트 → retry/크레딧. 어떤 경로도 stuck 없음.
- Bad Ending의 의식서 99조는 매체에 맞게 리라이트: 스크롤 문장 → 시간/대답 기반 문장 (귀신 질문 3번 모티프와 연결). 문안은 시나리오 수정 시 확정.
- 모든 엔딩 retry 시 사운드/상태 완전 정리 (구 YEO-18 Howl leak 교훈).

## 9. Open Questions

1. **Phase 2 R1~2에서 유저가 ✋(보)를 냈을 때의 연출.** 각본상 귀신이 ✊를 내고도 판정승을 가져간다 (귀신 주도 = 판정권도 귀신). "내가 이겼는데?"라는 유저 반발을 위화감 연출(귀신의 일방적 승리 선언 등)로 받아줄지, 귀신 손 표기를 조정할지 미정. → [4] 구현 전 확정 필요.
2. **귀신 질문 타이밍.** Q1(R1 후)·Q2(R2 후)는 확정. Q3 "지금 무서워?"의 삽입 지점 (ROUND3_SETUP 전후 어디인지) 미정.
3. **Bad Ending 99조 문안.** 방향(시간/대답 기반)은 확정, 문장은 시나리오 리라이트 시.
4. **ROUND3_SETUP 독백 대사.** 인과 명시 방침만 확정, 문안은 placeholder (결정 2).
